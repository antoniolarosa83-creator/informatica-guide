CONOSCERE SUPABASE

☁️ Che cos'è Supabase?

Supabase è una piattaforma cloud che ti dà un database PostgreSQL completo, gestibile dal browser, con:

  • un piano gratuito perfetto per la scuola
  • una GUI (interfaccia grafica) per creare tabelle senza codice: il Table Editor
  • un SQL Editor per quando vorrai scrivere query "vere"
  • servizi extra (autenticazione, API): per progetti futuri!

Sito ufficiale: supabase.com

🔎 PostgreSQL chi? È uno dei DBMS relazionali più potenti e usati al mondo, open source, nato all'università di Berkeley. Supabase te lo serve "pronto all'uso" nel cloud: tu pensi allo schema, lui pensa ai server.
💡 Perché ci piace per la didattica: tutto ciò che abbiamo studiato — tabelle, PK, FK, vincolo referenziale, politiche CASCADE — in Supabase si vede e si clicca. La teoria diventa un laboratorio.

LO SCHEMA LOGICO

📐 Il progetto da realizzare

Costruiremo lo schema logico "scolastico" derivato con le regole del modello relazionale:

Classe(id, nome, aula) Studente(id, cognome, nome, data_nascita, classe_id*) ← 1:N Corso(id, titolo) Iscrizione(studente_id*, corso_id*, voto) ← ponte N:N
📌 Convenzioni "da database reale": nomi di tabelle e colonne in minuscolo, senza spazi né accenti, con l'underscore: data_nascita, classe_id. Te le ritroverai identiche in SQL.
💡 Riconosci la teoria? Classe–Studente è la 1:N (FK nel lato N) e Iscrizione è la tabella ponte della N:N Studente–Corso, con l'attributo di associazione voto. Tutto torna!

VINCOLO REFERENZIALE

⚖️ Le "Action": cosa fare se il padre sparisce?

Quando crei la FK, la GUI chiede: "Action if referenced row is removed". È la politica di reazione della teoria!

Opzione GUISignificatoEsempio Classe→Studente
Restrict / No actionvieta la cancellazione se ci sono figlinon puoi cancellare la 4B finché ha studenti ✋
Cascadecancella a cascata anche i figlicancelli la 4B → spariscono anche i suoi studenti! 💥
Set NULLi figli restano, con FK svuotatagli studenti restano "senza classe" (classe_id = NULL)
Set defaultFK impostata a un valore predefinitogli studenti passano alla classe "in attesa"
💡 Per il nostro schema scolastico: scegli Restrict per Classe→Studente (prudenza!) e Cascade per Studente→Iscrizione (senza studente, le sue iscrizioni non hanno senso).
📌 Nei compiti scritti: saper motivare la scelta della politica vale quanto la FK stessa. Non esiste la scelta "sempre giusta": esiste quella giusta per quella realtà.

VINCOLO REFERENZIALE

🚨 L'esperimento: proviamo a violare il vincolo!

Esperimento 1 — FK inesistente

Inserisci uno studente con classe_id = 99 (nessuna classe ha id 99)…

❌ ERROR: insert or update on table "studente" violates foreign key constraint "studente_classe_id_fkey"

Il DBMS rifiuta la riga: il vincolo referenziale ti ha appena salvato da un dato orfano.

Esperimento 2 — cancellare il padre

Prova a cancellare la classe 4B mentre ha studenti (con ON DELETE Restrict)…

❌ ERROR: update or delete on table "classe" violates foreign key constraint... Key (id)=(2) is still referenced from table "studente"

Anche qui: bloccato. Sposta prima gli studenti (o scegli un'altra politica), poi cancella.

📌 Questo esperimento È la lezione: hai visto con i tuoi occhi la teoria del modello relazionale diventare un guardiano automatico dei tuoi dati. Fallo sempre vedere anche nei tuoi progetti d'esame!
LO SCHEMA LOGICO

🌉 Passo 7: completare lo schema (la tabella ponte)

1
Crea la tabella corso: id (PK), titolo (text).
2
Crea la tabella iscrizione con: studente_id (int8), corso_id (int8), voto (int4, Nullable: il voto arriva dopo!).
3
Aggiungi due FK: studente_id → studente.id (Cascade) e corso_id → corso.id (Cascade).
4
Rendi la PK composta: nella creazione tabella spunta Primary su entrambe le colonne studente_id e corso_id (e rimuovi la colonna id proposta).
iscrizione(studente_id*, corso_id*, voto)
📌 La PK composta nella GUI: puoi selezionare più colonne come Primary nella finestra di creazione della tabella: insieme formeranno la chiave composta. Proprio come da teoria: la coppia (studente, corso) non si ripete.
💡 Verifica finale: inserisci un'iscrizione con lo stesso studente e lo stesso corso due volte: il DBMS rifiuta la seconda (violazione PK). Tre vincoli su tre, testati! 🏆

VERSO SQL

📝 Sbircia dietro la GUI: il codice SQL

Tutto quello che hai cliccato, Supabase lo ha tradotto in SQL. Nel SQL Editor avresti potuto scrivere:

create table studente ( id bigint generated by default as identity primary key, cognome text not null, nome text, data_nascita date, classe_id bigint references classe(id) on delete restrict );

Riconosci tutto?

  • primary key → la PK ✓
  • not null → attributo obbligatorio ✓
  • references classe(id) → la FK, cioè il vincolo di integrità referenziale ✓
  • on delete restrict → la politica di reazione ✓
📌 GUI e SQL sono due facce della stessa cosa. La GUI è perfetta per imparare; l'SQL sarà il passo successivo del corso — e ora sai già cosa significa ogni riga.

ESERCITAZIONE

💪 Esercizi proposti: porta i tuoi schemi nel cloud

1 La società sportiva: realizza su Supabase lo schema Squadra(1)–Atleta(N). FK con ON DELETE Restrict. Popola e prova a cancellare una squadra con atleti.
🔍 Aiuto
Tabelle: squadra(id, nome, campionato) · atleta(id, nome, cognome, data_nascita, squadra_id*). La violazione da mostrare: DELETE su una squadra con atleti → errore Restrict.
2 La biblioteca: Libro–Prestito(N:N con date)–Studente. Decidi tu: la PK di prestito può essere la coppia (libro, studente)? O serve anche data_inizio? Motiva!
🔍 Aiuto
Se lo studente può riprendere lo stesso libro in momenti diversi, la coppia non basta: PK = (libro_id, studente_id, data_inizio). In Supabase: tre colonne spuntate come Primary. È la stessa riflessione della palestra sul modello relazionale!
3 La pizzeria (sfida): 6 tabelle, 2 ponti, FK a raffica. Scegli le politiche ON DELETE motivandole nella relazione (cosa succede se cancello una pizza dal menù? E un cameriere?).
🔍 Aiuto
Suggerimenti: pizza cancellata → Cascade su "contiene" (gli ingredienti della pizza sparita non servono) ma Restrict su "include" se ci sono ordinazioni storiche da conservare. Il cameriere licenziato? Set NULL sulle sue ordinazioni o Restrict: discutine, non c'è una sola risposta giusta.
